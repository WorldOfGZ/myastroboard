# ⚙️ Update

This guide covers all methods of updating MyAstroBoard.

## Updating

For users running the published image:
```bash
# Pull latest version
docker compose pull

# Restart with new version
docker compose up -d

# View current version
docker exec myastroboard cat /app/VERSION
```

For developers:
```bash
# Update repository
git pull

# Rebuild and restart
docker compose down
docker compose up -d --build

# Remove previous images
docker image prune -f
```

## Version History

See [Releases](https://github.com/WorldOfGZ/myastroboard/releases) for version history and changelogs.

## Backup Before Updating

Your data is persisted in volumes and will not be lost during updates. However, to be safe:

```bash
# Backup configuration
cp -r data data.backup

# Backup outputs (specific if you want kept these files, but they are regenerated)
cp -r uptonight_outputs uptonight_outputs.backup
```

## Rollback to Previous Version

If you need to rollback:

```bash
# For specific version
docker compose down
docker pull ghcr.io/worldofgz/myastroboard:v1.0.0
docker compose up -d
```

## Note for developers

### Uptonight Version

The uptonight Docker image version is managed by the constant `UPTONIGHT_VERSION` in the constant file `backend/constants.py`. This constant contains only the version tag (e.g., `2.5`).

The system will use `mawinkler/uptonight:{UPTONIGHT_VERSION}` defined in constant `UPTONIGHT_IMAGE`. This constant must remain unchanged.

> [!CAUTION]
> This repo is based on a defined version of UpTonight, increase version needs certainly other modifications than a version change. Like New catalogue for example.

### Adding New Target Catalogues

When UpTonight adds a new target catalogue (e.g., `newCatalogue.yaml`), the catalogue list need to be changerd.

The catalogue list is managed by the constant `UPTONIGHT_CATALOGUES` in the constant file `backend/constants.py`. This constant contains list of the catalogues, without `.yaml` extension (e.g., `['GaryImm', 'Herschel400',... , 'Pensack500']`).

Then add the YAML file to `targets/` and run the analysis script:

1. Place the catalogue file in `targets/`:
   ```
   targets/newCatalogue.yaml
   ```
2. Run the script from the repository root:
   ```bash
   python scripts/analyse_catalogues.py
   ```
3. The script automatically:
   - Generates `backend/catalogues.json` from all YAML files present in `targets/`
   - Regenerates `backend/catalogue_aliases.json` for cross-catalogue object matching
4. Save and commit updated files
